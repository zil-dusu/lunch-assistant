## 2. 后端设计

### 2.1 分层架构

```
Controller 层   ← 接收 HTTP 请求，参数校验，调用 Service
    │
Service 层      ← 核心业务逻辑（推荐、收藏、排序）
    │
Repository 层   ← 数据访问（JPA）
    │
Entity 层       ← 实体类（映射数据库表）
```

### 2.2 实体设计

**Food（菜品）**

```java
@Entity
public class Food {
    Long id;           // 主键，自增
    String name;       // 菜品名称，非空
    String category;   // 分类：米饭/面食/小吃
    String spicy;      // 辣度：辣/不辣
    String source;     // 校园送标识
    Integer hotCount;  // 热度值，默认0
}
```

**Favorite（收藏）**

```java
@Entity
@IdClass(FavoriteId.class)
public class Favorite {
    String userId;     // 用户UUID（联合主键）
    Long foodId;       // 菜品ID（联合主键）
}
```

### 2.3 核心算法设计

**LRUCache（HashMap + 双向链表）**

- 设计目标：O(1) 时间内实现 get/put，容量满时淘汰最久未使用
- 应用场景：缓存用户近期推荐过的菜品，实现"换一个"防重复

```
HashMap<Key, Node>
   ↓
哨兵头 ←→ Node1 ←→ Node2 ←→ Node3 ←→ 哨兵尾
          (最近)             (最旧)
```

**归并排序**

- 时间复杂度：O(n log n)
- 空间复杂度：O(n)
- 特点：稳定排序
- 实现方式：递归分治 + 双指针合并

**TopKHeap（小顶堆）**

- 维护大小为 K 的最小堆
- 新元素 > 堆顶时替换并下沉
- 时间复杂度：O(N log K)，空间 O(K)

## 3. 前端设计

### 3.1 路由设计

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | Home.vue | 首页（推荐+筛选+动画） |
| `/favorites` | Favorites.vue | 收藏列表 |
| `/hotrank` | HotRank.vue | 热度榜 |

路由模式：Hash 模式（`#/favorites`、`#/hotrank`）

### 3.2 组件设计

**Home.vue（首页）**
- 分类筛选按钮组（动态渲染）
- 辣度筛选按钮组（动态渲染）
- 滚动抽选动画区域
- 推荐结果卡片（emoji + 标签 + 按钮）
- 底部 TabBar 导航

**Favorites.vue（收藏页）**
- 收藏列表（左滑删除）
- NavBar 返回
- 底部 TabBar 导航

**HotRank.vue（热度榜）**
- 排名列表（前3名特殊样式）
- NavBar 返回
- 底部 TabBar 导航

### 3.3 状态管理

无全局状态管理（Vuex/Pinia），采用 **localStorage + 组件内 ref** 方案：
- 用户 UUID：localStorage 持久化
- 筛选状态：组件内 ref
- 收藏状态：通过 API 实时同步

## 4. 数据库设计

### 4.1 ER 图

```
┌──────────┐          ┌──────────┐
│   Food   │          │ Favorite │
├──────────┤          ├──────────┤
│ id (PK)  │◄─────────│ food_id  │
│ name     │          │ user_id  │
│ category │          └──────────┘
│ spicy    │               │
│ source   │               │ 联合主键
│ hot_count│               │ (user_id, food_id)
└──────────┘               │
                    user_id 来自前端 UUID
```

### 4.2 建表语句（JPA 自动生成）

```sql
CREATE TABLE food (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    category VARCHAR(50),
    spicy VARCHAR(10),
    source VARCHAR(100),
    hot_count INT DEFAULT 0
);

CREATE TABLE favorite (
    user_id VARCHAR(64),
    food_id BIGINT,
    PRIMARY KEY (user_id, food_id)
);
```

## 5. API 设计

### 5.1 接口规范

| 项目 | 规范 |
|------|------|
| 基础路径 | `/api` |
| 请求格式 | JSON |
| 响应格式 | JSON |
| 字符编码 | UTF-8 |

### 5.2 接口列表

| 方法 | 地址 | 请求体 | 响应 |
|------|------|--------|------|
| GET | `/api/foods` | query: category?, spicy? | `Food[]` |
| GET | `/api/categories` | — | `String[]` |
| GET | `/api/spicies` | — | `String[]` |
| POST | `/api/recommend` | `{userId, category?, spicy?, fromFav?, excludeId?}` | `Food` 或 `{error}` |
| POST | `/api/favorite/toggle` | `{userId, foodId}` | `{foodId, favorited, message}` |
| GET | `/api/favorites` | query: userId | `Food[]` |
| GET | `/api/hotrank` | — | `Food[]` (Top5) |
| GET | `/api/algorithm/sort` | query: by, desc | `{algorithm, timeMillis, data}` |
| GET | `/api/algorithm/topk` | query: k, by | `{algorithm, k, data}` |

### 5.3 错误处理

- 推荐无结果：返回 400 + `{"error": "没有符合条件的菜品！"}`
- 无收藏推荐：返回 400 + `{"error": "您还没有收藏菜品，请先添加最爱！"}`
