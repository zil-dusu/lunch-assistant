# 午餐决策助手 — 需求文档

## 1. 项目背景

大学生每天面临"中午吃什么"的选择困难。本项目通过随机推荐、分类筛选、收藏管理等功能，帮助用户快速决策，减少纠结时间。

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 普通用户 | 浏览菜品、随机推荐、管理收藏、查看热度榜 |

> 本项目无需登录注册，通过浏览器 localStorage UUID 识别用户身份。

## 3. 功能需求

### 3.1 菜品浏览

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-01 | 菜品列表 | 展示所有菜品的基本信息（名称、分类、辣度、来源、热度） | P0 |
| F-02 | 分类筛选 | 按米饭/面食/小吃筛选菜品 | P0 |
| F-03 | 辣度筛选 | 按辣/不辣筛选菜品 | P0 |

### 3.2 随机推荐

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-04 | 随机推荐 | 从符合条件的菜品中随机选一道，显示推荐结果卡片 | P0 |
| F-05 | 最爱推荐 | 从用户收藏中随机推荐一道 | P1 |
| F-06 | 换一个 | 重新随机推荐，LRU缓存保证近期不重复 | P0 |
| F-07 | 推荐动画 | 滚动抽选动画效果，增强交互体验 | P2 |

### 3.3 收藏管理

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-08 | 收藏切换 | 一键收藏/取消收藏菜品 | P0 |
| F-09 | 收藏列表 | 查看所有已收藏菜品 | P0 |

### 3.4 热度榜

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-10 | 热度排行 | 按推荐次数展示 Top 5 菜品 | P0 |

### 3.5 算法演示

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-11 | 归并排序演示 | 展示归并排序 O(n log n) 对菜品排序及耗时 | P2 |
| F-12 | Top-K 堆演示 | 展示小顶堆 O(N log K) 取热度前K | P2 |

## 4. 非功能需求

| 编号 | 需求 | 描述 |
|------|------|------|
| NF-01 | 响应速度 | API 响应时间 < 2 秒 |
| NF-02 | 移动适配 | 适配手机屏幕（最大宽度 480px） |
| NF-03 | 数据持久化 | 菜品数据持久化存储，重启不丢失 |
| NF-04 | 用户隔离 | 不同浏览器用户的收藏数据互不影响 |

## 5. 数据需求

### 5.1 菜品数据

每条菜品包含以下属性：

| 属性 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 名称 | 文本 | 菜品名称 | "港式烧腊扒饭" |
| 分类 | 枚举 | 米饭 / 面食 / 小吃 | "米饭" |
| 辣度 | 枚举 | 辣 / 不辣 | "辣" |
| 来源 | 文本 | 支持校园送/不支持校园送 | "支持校园送" |
| 热度 | 整数 | 被推荐次数 | 15 |

初始数据：21 道真实校园周边菜品。

## 6. 验收标准

| 场景 | 预期结果 |
|------|---------|
| 用户首次打开页面 | 看到空状态提示，点击按钮开始推荐 |
| 用户选择"米饭"+"辣"后推荐 | 返回的菜品分类为米饭、辣度为辣 |
| 用户点击"换一个" | 返回不同菜品，短期内不重复 |
| 用户收藏某菜品后再次进入 | 该菜品仍在收藏列表中 |
| 多次推荐同一菜品 | 该菜品热度值增加，热度榜排名上升 |
